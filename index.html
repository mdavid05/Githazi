Git

A Git célja az adott projekt menedzselése, ill. az adatok változásának nyomon követése. Ezen információk adatstruktúrákban történő tárolását repository-nak, röviden repo-nak, avagy lokális adatbázisnak nevezik.
Snapshot egy adott pillanatban, időpontban a könyvtárak, fájlok aktuális állapotát, tartalmát, verzióját jelenti.
Folyamata:
  • git init
  • git add .
  • git status
  • git commit -m "komment"
  • git pull
  • git remote add origin github link
  • git push --set-upstream origin master
init: Ez létrehozza a könyvtárunkban a .git/ adatbázist.
add: Az add parancs a módosult adatokat az index-be helyezi, snapshot-ot készít róluk.
status: Az index tartalmának és a working directory állapotának megjelenítésére szolgál.
commit: A módosítások eltárolására szolgál a lokális adatbázisban. Minden commit rendelkezik egy rövid szöveggel, leírással, mely arra utal, hogy milyen módosításokat tartalmaz.
pull: A központi szerverre felküldött változtatások, commit-ok letöltése.
remote: A távoli, szerveren lévő központi repo-t jelenti, mely segítségével tartják a kapcsolatot egymással a fejlesztők; origin névvel hivatkoznak rá.
push: Feltölti a módosított adatbázist a Szerverre.
